USE CASES

What it actually does for your workflow.

Parallel-session audible status

Run four Claude sessions across four projects. The computer tells you which one finished, in which repo, without you switching windows. Utterances queue sequentially through a shared lock so two sessions never speak over each other.

Ambient awareness

Look away from the terminal. The work narrates itself. Background music crossfades between tracks, ducks during voice, and restores on the next turn — you keep flow without staring at a scrollback.

Mic-aware halt

Start dictating into chat speech-to-text and the voice cuts itself off automatically. Detected via active source-output sample-rate, not a heuristic — peak meters and visualizers don't trigger it.

Project briefings on demand

Click "Trigger briefing" in the LCARS panel and Majel reads a status / mission / open-faults / recommended-actions report on any of your Claude projects. Persisted as markdown for diff-comparison later.

INSTALL

Five-minute setup.

  1. Clone the repository.
  2. Create a Python venv and install requirements.
  3. Drop your Anthropic key into ~/.majel_config.json.
  4. Wire hook.sh as a Claude Code Stop hook and session_start.sh as the SessionStart hook.
  5. Launch the LCARS control panel.
# clone + venv
git clone https://github.com/Jack-Taylor-Gorman/computerVoice.git
cd computerVoice
python3 -m venv venv
./venv/bin/pip install -r trekdata/requirements.txt

# add your Anthropic key (or skip for offline mode)
cat > ~/.majel_config.json <<'EOF'
{"voice_enabled": true, "voice_mode": "api",
 "anthropic_api_key": "sk-ant-...", "backend": "f5"}
EOF

# wire hooks in ~/.claude/settings.json:
#   Stop          → hook.sh
#   SessionStart  → session_start.sh

# launch the panel
./venv/bin/python majel_gui.py
Tested on Linux with PulseAudio / PipeWire. macOS and Windows are unverified. CUDA-capable GPU recommended for the F5 daemon (CPU fallback works but is slower).
